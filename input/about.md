# About this Pipeline

This site is generated by a tiny Node.js script that converts Markdown into HTML styled with the Daily Brief design system.

## Pipeline at a glance

1. Drop a `.md` file into the `input/` directory.
2. (Optional) Add YAML front-matter to set the page title, eyebrow label, meta line, and date.
3. Run `npm run build` — every `.md` becomes a styled `.html` in `dist/`, plus an `index.html` listing them all.

> **Tip:** Front-matter is optional. If you skip it, the first `# Heading` becomes the title and today's date fills the meta line.

## Front-matter fields

| Field   | Purpose                                | Default                |
|---------|----------------------------------------|------------------------|
| title   | Page title (header H1 and `<title>`)   | First H1 in the file   |
| eyebrow | Small uppercase label above the title  | `Document`             |
| meta    | Subtitle line below the title          | Today's date           |
| date    | Used to sort entries on the index page | File mtime             |
| summary | One-line description on the index page | First paragraph        |

## Local development

```bash
npm install
npm run build           # one-shot build of input/ -> dist/
npm run build:watch     # rebuild on input/ or template change
```

## Deployment

A GitHub Action runs on every push to `main`: it installs dependencies, builds `dist/`, and deploys it as the GitHub Pages artifact. No generated files are committed.
